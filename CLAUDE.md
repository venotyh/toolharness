## Architecture

```
rawInput (unknown)
  │
  ├─ applyRelationalDefaults()   inject missing co-dependent fields
  │
  ├─ schema.safeParse()          first validation attempt (Zod)
  │     success → execute()
  │
  ├─ repair()                    walk ZodError issues, apply shape repairs
  │     in order: #2 stringify_parse → #4 string_to_array
  │               → #1 null_drop (optional fields only)
  │               → #3 object_to_array
  │
  ├─ schema.safeParse()          second validation attempt
  │     success → execute()      + logger.info tool_input_repaired
  │
  └─ formatFailureMessage()      model-readable string, no "Error:" prefix
                                 + logger.warn tool_input_invalid
```

Key invariant: **valid inputs are never touched** — repair only runs at paths the schema rejected.

## Key decisions

- **Zod only** — no adapter layer for JSON Schema or other validators
- **Anthropic SDK only** — `wrap()` produces `Anthropic.Tool`; `input_schema` is derived from the Zod schema via `zod-to-json-schema`
- **null_drop is optional-only** — dropping null from a required field changes "wrong type" to "missing required" with no benefit; the error message is more useful left intact
- **Repair order matters** — #2 (stringify_parse) must precede #4 (string_to_array) or `'["a","b"]'` becomes `['["a","b"]']`
- **Return, don't throw** — `execute()` returns a model-readable string on unrecoverable failure; callers never need try/catch
